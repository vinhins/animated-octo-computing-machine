# MetaQuote API Protocol - Reverse Engineering Analysis

## Summary

Successfully reverse-engineered the HTTP API call from the MetaTrader 4 Android app by analyzing decompiled Smali code. The app uses libmt4.so to sign requests to MetaQuote broker servers.

## API Protocol

### Endpoint (CONFIRMED)
```
POST https://api{1-15}.cdnfx.net/public/mt4/network/mobile
```

The app tries servers 1-15 in sequence as failover mechanism.

### Request Format
```
company={broker_name}&signature={hex_signature}
```

### Signing Algorithm

1. **Build signing string**: `company={broker_name}&code=mt4`
2. **Sign with libmt4.so**: Call `libmt4.so.generate(signing_string)` → returns hex signature
3. **Build request body**: `company={broker_name}&signature={signature_hex}`

### Example Exchange

**Input broker name**: `ICMarkets`

**Signing input**:
```
company=ICMarkets&code=mt4
```

**Signature output** (hex, from libmt4.so):
```
a3f2b1c5d80e2f4c9a8b7e6d5c4b3a2f1e0d9c8b7a6f5e4d3c2b1a0f9e8d7
```

**HTTP POST body**:
```
company=ICMarkets&signature=a3f2b1c5d80e2f4c9a8b7e6d5c4b3a2f1e0d9c8b7a6f5e4d3c2b1a0f9e8d7
```

**Response**: JSON list of available servers/accounts from MetaQuotes

- **Location**: [e6.smali line 189-227](../decoded_apks/v400.1456_301/smali/e6.smali#L189)

  ```smali
  invoke-static {v0, v1, v2}, Ljava/lang/String;->format(...)
  ```
- **Returns**: `"https://api{1}.cdnfx.net/"` through `"https://api{15}.cdnfx.net/"`
- **Line 1075+**: `a()` method - main execution flow
- **Line 13-15**: Creates BrokerSignature object from libmt4.so
- **Line 18-20**: Wraps it with rk2$a builder
- **Line 29**: Calls the request building logic
- **Line 33**: Makes HTTP POST request

### File: rk2$a.smali (Request building & signing)
- **Line 33**: Format string with code parameter: `"company=%s&code=mt4"`
- **Line 41**: **CRITICAL** - calls `Lxq2::a(signing_string)` interface
  - This interface is implemented by BrokerSignature from libmt4.so
  - Takes the signing string, returns signature hex
- **Line 52**: Final format string: `"company=%s&signature=%s"`
  - First %s: broker name
  - Second %s: signature from libmt4.so

## Key Data Flow

```
ServersListFragment (UI)
  ↓ onTextChanged(query_text)
ViewModel ao2
  ↓ D(query_text) - debounce 300ms
Repository ih
  ↓ d(query, callback) - concurrent execution
Worker rk2 (Callable)
  ↓ a() - main execution
  ├─ Create BrokerSignature wrapper for libmt4.so
  ├─ Build signing string "company=X&code=mt4"
  ├─ Call libmt4.so to get signature
  ├─ Build POST body "company=X&signature=SIG"
  └─ POST to MetaQuotes API
    ↓ Response: broker list
TerminalServers
  └─ Filter and cache results
```

## Headers Used

From rk2.smali:
- **User-Agent**: Generated by `Lg11::a()` and encoded by `Lg11::d()`
- **Cookie**: Built as `x80` builder with key "server" containing terminal network info
- **Content-Type**: `application/x-www-form-urlencoded` (implicit from POST body format)

## Important Notes

1. **Signature is deterministic**: Same input always produces same signature
2. **Signature includes "code=mt4"**: But this parameter doesn't appear in the final POST body
3. **Network location required**: Broker name must be URL-decoded correctly
4. **Session context**: Terminal also includes User-Agent and Cookie with network context
5. **Cross-platform works**: Compiled .class files on x86 work on ARM64 phone
